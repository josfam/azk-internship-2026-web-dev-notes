## 2.0. A note about Windows PowerShell execution permissions for Windows users

On Windows, PowerShell may block package-manager commands such as `npm`, `pnpm`, or `yarn` because they run PowerShell script files like `npm.ps1` or `pnpm.ps1`.

Run the following command in PowerShell so that you will not run into any disturbances when we get to the software installation/project setup part

```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

That command in PowerShell, for example:
![PowerShell command for setting execution policy](https://raw.githubusercontent.com/josfam/azk-internship-2026-web-dev-notes/refs/heads/main/Web%20development%2C%20week%202/images/2.0.1.set%20execution%20policy.png)

You can even check that this policy has indeed been set for your PowerShell user by running this command:

```powershell
Get-ExecutionPolicy -List
```

On my machine, this shows the following output:
![Execution Policiy list terminal output](https://raw.githubusercontent.com/josfam/azk-internship-2026-web-dev-notes/refs/heads/main/Web%20development%2C%20week%202/images/2.0.1.check-that-the-policy-is-set.png)

`CurrentUser` shows `RemoteSigned`, which means that the commands were successful.
